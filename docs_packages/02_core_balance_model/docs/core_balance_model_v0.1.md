# 《核心数值模型 v0.1》

项目：双人雷霆战机修仙版  
版本：v0.1  
适用范围：第一大阶段「青云山·妖潮初临」、局内战斗垂直切片、Codex 首版实现  
文档目标：给出可实现、可测试、可调参的第一批核心数值公式与预算表。

---

## 0. 核心结论

v0.1 的数值地基必须围绕两条独立成长线搭建：

| 成长线 | 程序状态名 | 设计含义 | 获取节奏 | 主要来源 | 触发结果 |
|---|---|---|---|---|---|
| 灵气经验 | `TeamInsightExpState` | 法的广度与深度 | 高频 | 小怪、灵气球、怪潮、精英灵气 | 时停顿悟、三选一、Build 构筑 |
| 修为 | `PlayerCultivationState` | 生命本质与境界 | 低频/关键 | 周天吐纳、丹药、天材地宝、精英/Boss | 层数突破、境界瓶颈、雷劫、精气神质变 |

经验值绝不直接提升境界；修为也不直接触发肉鸽抓牌。  
经验值服务“法术/天赋/功法/体质/灵宝”的局内 Build；修为服务“练气、筑基、金丹”等境界层级和雷劫。

## 1. 数值设计原则

### 1.1 第一大阶段必须先爽，后硬核

第一大阶段不是惩罚玩家的地方，而是验证核心循环的地方。v0.1 的第一阶段应达成：

- 普通小怪 0.3–0.8 秒死亡。
- 冲锋怪 0.6–1.2 秒死亡。
- 远程弹幕怪 1.0–1.8 秒死亡。
- 重装怪 2.5–4.5 秒死亡。
- 精英怪 10–18 秒死亡。
- Boss 90–130 秒死亡。

### 1.2 玩家前 5 分钟应该获得 3 次明确构筑变化

第一大阶段 5–7 分钟内，推荐触发：

- 第 1 次顿悟：1-1 结束，快速建立爽感。
- 第 2 次顿悟：1-3 结束，开始成型。
- 第 3 次顿悟：1-4 结束，进入 Boss 前形成主要 Build。
- 第 4 次顿悟：Boss 后或高根骨/高难度条件下触发。

### 1.3 雷劫不是随机秒杀，而是高压考试

局内动态雷劫应满足：

- 有 0.9–1.1 秒清晰预警。
- 单次命中很痛，但不应满血秒杀。
- 不能被普通清弹法术摧毁。
- 成功后必须给出极强正反馈：全屏清场、满状态、境界质变、金光表现。

### 1.4 双人模式不能简单把所有数值翻倍

两名玩家并不等于两倍输出稳定命中。双人会增加画面复杂度、拉人风险、资源协商和走位干扰。因此 v0.1 推荐：

- 普通怪生命：`1.35x–1.45x`
- 精英生命：`1.55x–1.70x`
- Boss 生命：`1.55x–1.65x`
- 敌弹密度：`1.10x–1.18x`
- 阶段灵气经验总量：`1.10x–1.18x`

## 3. 玩家基础面板

### 3.1 v0.1 裸装基准

练气一层、无局外强化、无功法、无灵宝的单人裸装基准：

| 属性 | 值 | 说明 |
|---|---:|---|
| 精 `jing` | 10 | 体魄基准 |
| 气 `qiRoot` | 10 | 真元基准 |
| 神 `shen` | 10 | 神识基准 |
| 悟性 `comprehension` | 10 | 奖励质量基准 |
| 气运 `fortune` | 10 | 掉落/奇遇/重 Roll 基准 |
| 根骨 `constitution` | 10 | 灵气经验转化基准 |
| 业力 `karma` | 0 | 高风险收益基准 |
| 生命上限 `maxHp` | 100 | 约 6–8 次普通弹容错 |
| 真元上限 `maxQi` | 100 | 可释放 2 次中等法术 |
| 移速 `moveSpeed` | 400 | 适配宽屏 STG |
| 判定半径 `hitboxRadius` | 7 | 不随神识变化 |
| 拾取半径 `pickupRadius` | 90 | 神识可提升 |
| 基础暴击率 | 5% | 神识可提升 |
| 暴击倍率 | 1.5x | 后续可被剑修/魂修提高 |
| 基础穿透/破甲 | 0% | 神识可提升 |

### 3.2 精气神换算公式

```ts
maxHp = 60 + jing * 4 + realmHpFlatBonus;
maxQi = 50 + qiRoot * 5 + realmQiFlatBonus;
moveSpeed = 400 * moveSpeedMultiplier;

passiveQiRegenPerSecond = 0.8 + qiRoot * 0.04;
cultivationRegenPerSecond = 0.9 + qiRoot * 0.04 + shen * 0.02;

pickupRadius = 70 + shen * 2 + treasurePickupBonus;
critChance = clamp(0.02 + shen * 0.003, 0.02, 0.35);
armorPierce = clamp((shen - 10) * 0.0025, 0, 0.25);
lockOnRange = 260 + shen * 10;
```

裸装代入：

```text
maxHp = 100
maxQi = 100
passiveQiRegen = 1.2/s
cultivationRegen = 1.6/s
pickupRadius = 90
critChance = 5%
lockOnRange = 360px
```

## 4. 伤害公式

### 4.1 统一伤害管线

```ts
finalDamage = baseDamage
  * sourcePowerMultiplier
  * playerAttackMultiplier
  * spellOrArtifactMultiplier
  * elementMultiplier
  * critMultiplier
  * armorMultiplier
  * difficultyDamageMultiplier;
```

其中：

```ts
critMultiplier = rng.roll(critChance) ? critDamageMultiplier : 1;

armorMultiplier = 1 - max(0, enemyArmor - playerArmorPierce);

armorMultiplier = clamp(armorMultiplier, 0.35, 1.0);
```

### 4.2 五行倍率 v0.1

v0.1 先做轻量五行，不做复杂抗性表：

| 情况 | 倍率 |
|---|---:|
| 属性克制 | 1.25x |
| 属性被克 | 0.85x |
| 属性相生连携 | 1.15x + 状态触发概率 |
| 无关系 | 1.00x |

## 5. 本命法宝 DPS 预算

v0.1 的本命法宝必须形成三种手感：飞剑单点、葫芦清场、重印压制。

### 5.1 外带本命法宝基准

| 法宝 | 攻击间隔 | 单发/单轮伤害 | 有效单体 DPS | 群体能力 | 设计目标 |
|---|---:|---:|---:|---|---|
| 青霜飞剑 | 0.16s | 11 | 68.8 | 中 | 标准强单体，Boss 稳定 |
| 紫阳葫芦 | 0.42s | 5×6 | 42–58 | 强 | 扇形清怪，Boss 较弱 |
| 玄岳重印 | 0.90s | 52 范围 | 55–64 | 中强 | 慢速高压，打精英舒服 |

说明：紫阳葫芦对同一目标有命中片数上限，避免贴脸 Boss 时爆炸超模。

### 5.2 局内第二本命法宝缩放

局内获得的第二本命法宝不能满额叠加，否则第一阶段中期直接数值失控。v0.1 推荐：

```ts
innerArtifactPower = 0.70 + innerArtifactLevel * 0.08;
innerArtifactPowerMax = 1.00;
```

也就是说：

| 局内本命法宝等级 | 输出倍率 |
|---:|---:|
| 1 | 70% |
| 2 | 78% |
| 3 | 86% |
| 4 | 94% |
| 5 | 100% |

```ts
passiveQiRegen = 0.8 + qiRoot * 0.04;
qiOrbGain = orbBaseQi * (1 + qiGainBonus);
coopQiEcho = pickedByOnePlayer ? 0.30 : 0;
```

## 8. 丹药数值预算

丹药必须要求提前预判，不能当即时血瓶。

| 丹药 | 消化时间 | 效果 | 设计目标 |
|---|---:|---|---|
| 回春丹 | 12s | 每秒恢复 8 HP，总计 96 | 提前续航，吃晚了救不了 |
| 燃血丹 | 8s | 普攻射速 +35%，法术伤害 +25% | Boss 爆发/精英压制 |
| 清心丹 | 6s | 解控，移速 +15%，受击硬直免疫 | 高压弹幕脱困 |
| 小破境丹 | 5s | 增加 120 修为，不影响灵气经验 | 推动境界瓶颈 |

燃血丹后遗症：

```ts
weaknessDuration = 3;
moveSpeedMultiplier = 0.82;
qiGainMultiplier = 0.75;
```

丹药叠加原则：

- 同类丹药不可同时消化。
- 不同类丹药可以同时生效，但 v0.1 建议最多 2 个消化效果并行。
- 丹药效果受 `pillEffectMultiplier` 影响，默认 1.0。

## 10. Boss 数值预算

第一 Boss「青云劫灵」目标时长：

| 模式 | HP | 目标战斗时长 |
|---|---:|---:|
| 单人普通 | 10500 | 90–120s |
| 双人普通 | 16500 | 90–130s |

| 双人困难 | 25500 | 105–145s |

Boss 阶段：

| 阶段 | 血量区间 | 目标时长 | 压力来源 |
|---|---|---:|---|
| Phase 1 | 100%–66% | 30–40s | 雷珠散射 + 山魈 |

2. 走位区域压缩。
3. 丹药/法术时机压力。

## 11. 灵气经验曲线

### 11.1 TeamInsightExpState

```ts
interface TeamInsightExpState {
  level: number;
  exp: number;
  expToNext: number;
  sharedFortuneReroll: number;
}
```

经验阈值采用“前快后慢”：

| 顿悟等级 | 本级所需灵气经验 | 推荐触发点 |
|---:|---:|---|
| 1 | 160 | 1-1 结束 |
| 2 | 420 | 1-3 结束 |
| 3 | 680 | 1-4 结束 |
| 4 | 900 | Boss 后或高根骨局 |
| 5 | 1200 | 第二大阶段 |
| 6 | 1580 | 第二/第三大阶段 |

### 11.2 根骨对灵气经验的影响

双人模式下，为避免抢灵气，灵气经验归团队。根骨使用团队平均值：

```ts
avgConstitution = average(alivePlayers.constitution);
teamInsightExpGain = baseExp * (1 + (avgConstitution - 10) * 0.015);
```

例如：

| 平均根骨 | 经验倍率 |
|---:|---:|
| 8 | 0.97x |
| 10 | 1.00x |
| 15 | 1.075x |
| 20 | 1.15x |
| 30 | 1.30x |

### 11.3 第一大阶段经验预算

| 小阶段 | 时长 | 经验预算 | 是否触发顿悟 |
|---|---:|---:|---|
| 1-1 山门试炼 | 55s | 170 | 是，第 1 次 |
| 1-2 林间妖影 | 60s | 210 | 否，积累 |
| 1-3 雾中邪修 | 60s | 270 | 是，第 2 次 |
| 1-4 妖潮压境 | 70s | 620 | 是，第 3 次 |
| 1-5 Boss | 90–130s | 650 | Boss 后可选第 4 次 |

规则：每个小阶段最多触发 1 次顿悟；若中途经验溢出，延迟到小阶段结尾或安全窗口触发。

## 13. 境界突破与雷劫数值

### 13.1 小层突破

小层突破不触发雷劫，只给轻量反馈：

```ts
minorBreakthrough = {
  maxHpMultiplier: 1.06,
  maxQiMultiplier: 1.05,
  baseDamageMultiplier: 1.015,
  instantHealPercent: 0.15,
  instantQiPercent: 0.15
}
```

### 13.2 大境界突破：练气 → 筑基

当玩家在局内达到练气九层瓶颈，并获得足够修为时，触发局内动态雷劫。

成功奖励：

```ts
majorBreakthroughQiToFoundation = {
  maxHpMultiplier: 1.80,
  maxQiMultiplier: 1.65,
  shenDerivedStatsMultiplier: 1.35,
  baseDamageMultiplier: 1.12,
  moveSpeedMultiplier: 1.05,
  fullHeal: true,
  fullQi: true,
  screenClear: true,
  fortuneBoonDuration: 20,
```

雷劫时间轴：

| 时间 | 事件 |
|---:|---|

- 双人同时神魂且无法救援：本局按团灭结算。

## 20. v0.1 结论

《核心数值模型 v0.1》的目标不是一次性精确平衡，而是把第一版可玩性的“正确范围”框住：

1. 小怪死得快，保证割草。
2. 精英和 Boss 够硬，迫使使用法术和丹药。
3. 灵气经验高频推动顿悟 Build。
4. 修为低频推动境界和雷劫。
5. 雷劫是高压质变，不是普通升级。
6. 双人倍率保留协作爽感，不用粗暴翻倍。
7. 所有数值必须可配置、可观测、可调试。

v0.1 第一大阶段如果能稳定做到“3 次顿悟、2 次小层突破、1 场 90 秒以上 Boss 战、至少 1 次涉险救援、Debug 可触发雷劫”，这个垂直切片就已经具备足够高的第一版可玩性。
