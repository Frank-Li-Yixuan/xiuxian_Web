# 双人雷霆战机修仙版 · 局内 UI/UX 信息架构 v0.1

本包用于承接《第一大阶段局内垂直切片 v0.1》《核心数值模型 v0.1》，将局内 HUD、顿悟界面、雷劫提示、救援状态、双人信息、法术/丹药/法宝槽位等全部转化为可实现的 UI/UX 契约。

核心原则：

- 灵气经验是 Build 进度，触发时停顿悟。
- 修为是境界进度，触发瓶颈/雷劫/突破。
- 二者在视觉、命名、音效、动效、提示文案上必须彻底分离。
- 中央战斗区必须始终优先于 HUD，任何 UI 不得遮挡判定点和关键弹幕。
- UI 不依赖外部 CDN、外部字体或外部图片资源；图标可先用 Canvas/SVG/程序化图形占位。
