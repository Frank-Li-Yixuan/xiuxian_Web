## 下一步最应该细化的文档

### 1. 《核心数值模型 v0.1》

优先级最高。

要定：

- 精、气、神如何换算到 HP、真元池、拾取半径、索敌范围、暴击、穿透。
- 灵气经验曲线。
- 修为曲线。
- 怪物血量与玩家 DPS。
- Boss 血量与阶段时长。
- 丹药恢复量与副作用。
- 法术真元消耗与冷却。
- 双人时怪物血量和掉落倍率。

没有数值模型，数据表只是名字好听；有了数值模型，第一版才会真的好玩。

### 2. 《局内 UI/UX 与信息架构 v0.1》

要定：

- 两条成长条怎么显示，避免玩家混淆。
- 法术槽、丹药槽、消化槽、真元池、境界条如何布局。
- 顿悟界面双人明牌怎么交互。
- 雷劫来临时如何提示，不让玩家误以为是普通 Boss 技能。
- 神魂、救援、阳神状态怎么显示。

### 3. 《战斗手感与视觉特效规范 v0.1》

要定：

- 弹速、移速、屏幕震动强度。
- 爆炸粒子层级。
- 清屏特效优先级。
- 法术颜色体系。
- 雷劫预警视觉。
- 不依赖外部资源的 Canvas 特效规范。

### 4. 《联机同步技术设计 v0.1》

要定：

- 本地双人和在线双人的统一输入模型。
- Seeded RNG。
- 哪些事件必须确定性。
- 哪些视觉效果可以非确定性。
- 顿悟时停如何同步。
- 雷劫触发如何同步。
- 断线、重连、暂停和房主迁移先不做，但接口要预留。

### 5. 《局外洞府最小闭环 v0.1》

要定：

- 藏经阁：功/法分离，多功同修降速。
- 聚灵阵：挂机修为收益。
- 炼丹房：局内丹药、永久属性丹、破境丹、丹毒。
- 炼器阁：本命法宝与灵宝解锁升星。
- 局内材料如何流回局外。
- 失败后 100% 带回哪些资源。

## 推荐顺序

1. 核心数值模型。
2. UI/UX 信息架构。
3. 联机同步技术设计。
4. 战斗手感与特效规范。
5. 局外洞府最小闭环。

原因：  
数值决定第一版是否好玩；UI 决定玩家是否看懂两条成长；联机同步决定后续能不能上线双人；特效决定爽感；局外系统最后承接资源。
