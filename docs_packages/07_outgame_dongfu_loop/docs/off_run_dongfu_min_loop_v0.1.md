# 《局外洞府最小闭环 v0.1》

项目：双人雷霆战机修仙版  
版本：v0.1  
定位：第一版可玩 Demo 的局外系统设计、数据契约与 Codex 实施依据  
目标：让玩家完成第一大阶段后，能在局外立即获得“变强、再来一局”的明确驱动力。

---

## 0. 文档定位

本文件不是完整宗门经营设计，也不是大型修仙模拟器终局方案。v0.1 只定义 **最小可玩局外闭环**。

v0.1 的局外目标是：

```text
局内战斗结束
  ↓
100% 带回本局基础资源
  ↓
进入洞府结算
  ↓
收取聚灵阵挂机收益
  ↓
炼丹 / 炼器 / 修功 / 升法
  ↓
调整下局 Loadout
  ↓
重新进入第一大阶段或下一阶段
```

v0.1 不追求大而全，而要保证玩家每一局结束后至少有 1–3 个可执行的成长动作。

---

## 1. 核心设计原则

### 1.1 死亡不是失败，而是回府闭关

游戏采用《哈迪斯》式资源闭环：

```text
玩家战死或通关后，局内已获得的基础资源 100% 带回洞府。
```

剧情包装：

```text
肉身崩解，道友真灵携战利品遁回洞府。
不是 Game Over，而是“归府闭关”。
```

v0.1 中不做死亡扣资源、不做装备损坏、不做随机没收材料。局外系统的职责是消解挫败，并把挫败转化为“马上变强”的消费冲动。

---

### 1.2 局外只服务局内，不做无关经营膨胀

v0.1 局外只保留五个模块：

| 模块 | 作用 | 对局内的影响 |
|---|---|---|
| 洞府总览 | 结算、收菜、下局准备 | 给玩家清晰下一步 |
| 藏经阁 | 修“功”、研“法” | 基础数值、流派底板、法术等级 |
| 聚灵阵 | 挂机修为与灵石 | 推动境界、经济基础 |
| 炼丹房 | 消耗灵草炼丹 | 永久属性、局内丹药、破境材料 |
| 炼器阁 | 消耗矿石妖丹炼器 | 本命法宝、灵宝升星与解锁 |
| 劫雷台 | 局外境界突破 | 大境界跨越、硬核考试 |

弟子、宗门外交、交易市场、灵田经营、建筑摆放、洞府装饰、拍卖行、NPC 好感度，全部推迟到 v0.2+。

---

### 1.3 “功”与“法”必须分离

局外藏经阁必须严格区分：

```text
功 = 内功心法 / 秘术 / 体质底板，主要加数值与流派被动。
法 = 外放神通 / 主动技能，进入局内占 4 个法术槽。
```

| 类别 | 是否占局内法术槽 | 局外作用 | 局内作用 |
|---|---:|---|---|
| 功 | 否 | 长线修炼、提升永久属性、流派底板 | 作为初始被动或可被顿悟临时强化 |
| 法 | 是 | 解锁/升阶主动法术 | J/K/L/I 主动释放，消耗真元 |

关键规则：

```text
修功不等于学法。
学法不等于升境界。
境界突破不由灵气经验触发。
```

---

### 1.4 局外修为与局内灵气经验继续保持双轨

局外系统必须继承已经确定的双轨成长：

| 轨道 | 所属场景 | 用途 | 不可混用规则 |
|---|---|---|---|
| 灵气经验 | 局内 | 触发顿悟三选一，服务 Build | 不用于境界突破 |
| 修为 | 局内/局外 | 推动练气、筑基、金丹等境界 | 不直接触发顿悟抓牌 |

局外聚灵阵产出的主要是：

```text
修为 + 灵石
```

不是局内灵气经验。

---

### 1.5 第一版局外必须“少点但有用”

每次回洞府，玩家应该看到 1–3 个高价值操作，而不是 20 个红点。

v0.1 推荐局外操作节奏：

| 回府时间 | 玩家可做的事 |
|---|---|
| 第一次战死 / 通关后 | 炼 1–2 颗局内丹药，收挂机，修一本功，可能解锁一个法术 |
| 第二到三次回府 | 本命法宝升 1 星，灵宝升 1 星，功法升到 2–3 级 |
| 第四到五次回府 | 解锁新法术或新灵宝，准备筑基材料 |
| 达到瓶颈后 | 进入劫雷台，进行局外突破关卡 |

## 2. 局外主循环

### 2.1 完整流程

```text
RunSettlementReceipt
  ↓
资源入库 ResourceWallet
  ↓
洞府总览显示“本次收获”与“推荐闭关事项”
  ↓
收取聚灵阵 IdleClaim
  ↓
可选：藏经阁修功 / 研法
  ↓
可选：炼丹房炼制局内丹药 / 永久丹 / 破境丹
  ↓
可选：炼器阁升星本命法宝 / 灵宝
  ↓
如果修为达瓶颈：劫雷台突破
  ↓
配置下局 Loadout
  ↓
开始下一局
```

---

### 2.2 首屏洞府总览

洞府总览不是装饰页，而是局外决策中枢。

必须显示：

```text
当前境界 / 修为进度
下局 Loadout 摘要
资源钱包
聚灵阵可领取收益
可升级项目列表
是否达到瓶颈
推荐下一步
```

推荐文案：

```text
道友归府
本次携回：下品灵石 x240、灵草 x18、玄铁精 x9、残破玉简 x2、雷髓 x1
聚灵阵已积蓄：修为 +96、下品灵石 +80
可闭关事项：
  1. 青霜飞剑可升至 2 星
  2. 可炼制回春丹 x2
  3. 《锐金诀》可修至 2 层
```

---

### 2.3 局外红点优先级

v0.1 红点不能乱弹。按照优先级排序：

| 优先级 | 条件 | 模块 | 原因 |
|---:|---|---|---|
| 1 | 已达瓶颈且可突破 | 劫雷台 | 最高价值质变 |
| 2 | 可装备明显更强 Loadout | 炼器阁/装备页 | 影响下一局手感 |
| 3 | 可炼制空槽丹药 | 炼丹房 | 影响下局容错 |
| 4 | 可收取挂机收益 | 聚灵阵 | 收菜反馈 |
| 5 | 可修功/研法 | 藏经阁 | 长线成长 |
| 6 | 可清丹毒 | 炼丹房/灵泉 | 负面状态处理 |

### 3.3 资源归属：单人、本地双人、在线双人

v0.1 避免资源分配争执。

#### 单人

所有资源进入玩家个人洞府。

#### 本地双人

默认使用同一个共享洞府 Profile：

```text
P1/P2 共用资源库、共用建筑、共用装备库。
```

原因：本地双打多为同屏娱乐，先降低复杂度。

#### 在线双人

每名玩家拥有自己的个人洞府 Profile。一次联机结算会生成两份个人 Receipt：

```text
同一局中产生的基础材料，P1/P2 各获得一份完整副本。
稀有掉落按个人气运修正独立结算。
```

v0.1 不做玩家交易、不做资源分赃、不做队长分配。

## 4. 结算系统

### 4.1 RunSettlementReceipt

局内结束后，不直接修改局外存档，而是生成一张结算凭据：

```ts
RunSettlementReceipt
```

它记录：

```text
runId
stageReached
bossKilled
baseRewards
bonusRewards
firstClearRewards
difficultyMultiplier
playerModifiers
```

局外系统应用 Receipt 后，才写入 Profile。

这样便于：

```text
调试
回放
联机断线补发
防止重复结算
未来云存档校验
```

---

### 4.2 100% 保留与倍率

基础规则：

```text
本局已拾取/已结算进背包的基础资源全部保留。
```

难度影响的是奖励倍率，不影响“是否保留”。

| 难度 | 基础资源倍率 | 稀有材料倍率 | 说明 |
|---|---:|---:|---|
| normal | 1.00 | 1.00 | 标准体验 |
| hard | 1.15 | 1.25 | 怪物更强 |
| tribulation | 1.30 | 1.60 | 雷劫频率/词条更高 |

---

### 4.3 第一大阶段结算预算

| 进度 | 预计资源 | 设计意图 |
|---|---|---|
| 死在 1-1/1-2 | 少量灵石、青灵草、小妖丹 | 至少能炼 1 颗回春丹或修一点功 |
| 死在 1-3/1-4 | 中量灵石、玄铁精、残破玉简 | 能进行一次小升级 |
| 击败青云劫灵 | 雷髓、灵玉、较多灵石、首通奖励 | 能明显改变下一局配置 |

第一大阶段普通通关推荐奖励：

```text
下品灵石 320–420
青灵草 18–28
清心草 4–8
玄铁精 10–16
小妖丹 8–14
灵玉 1–3
雷髓 1
残破玉简 2–4
```

### 5.2 推荐闭关事项算法

```ts
recommendedActions = evaluateOutgameProfile(profile).slice(0, 3)
```

推荐顺序：

1. 修为已满，缺材料可提示“炼破境丹”；材料齐则提示“前往劫雷台”。
2. 若丹药槽为空，推荐炼制回春丹。
3. 若本命法宝可升星，推荐炼器。
4. 若有残破玉简可解锁新功/法，推荐藏经阁。
5. 若聚灵阵收益接近上限，推荐收菜。

### 6.2.3 同修多本与修炼速度惩罚

玩家可以同时安排多本功修炼，但会降低单本效率。

定义：

```text
聚灵阵每分钟产出 TrainingPower。
玩家把 TrainingPower 分配给多本功。
同修数量越多，产生额外“灵气冲突损耗”。
```

v0.1 推荐公式：

```ts
basePower = idleCultivationPerMinute
concurrentCount = activeTrainingItems.length
conflictLoss = getConflictLoss(activeTrainingItems)
parallelPenalty = 1 - max(0, concurrentCount - 1) * 0.10
finalPower = basePower * parallelPenalty * (1 - conflictLoss)
itemPower = finalPower * item.allocationWeight
```

推荐惩罚：

| 同修本数 | 基础并修效率 | 说明 |
|---:|---:|---|
| 1 | 100% | 单点突破最快 |
| 2 | 90% | 两本分配，总效率轻微损耗 |
| 3 | 80% | 可接受但明显变慢 |
| 4 | 70% | v0.1 上限，水桶号代价明显 |

元素冲突：

| 组合 | 额外损耗 |
|---|---:|
| 金 + 木 | 8% |
| 水 + 火 | 10% |
| 土 + 水 | 6% |
| 同元素 | 0% |
| 相生组合 | -5%，即效率奖励 |

v0.1 最大同修数建议为 3，Debug 可开到 4。

## 6.3 法：主动神通与局内槽位

### 6.3.1 法的规则

法是主动技能，进入局内占用 4 个法术格：

```text
J / K / L / I
```

局外研法带来：

```text
解锁法术
提高法术永久熟练度
提高可带入初始等级
解锁法术词条
```

局内顿悟带来：

```text
当局临时获得法术
当局临时升级法术
当局触发法术进化
```

---

### 6.3.2 局外等级与局内等级

必须区分：

| 概念 | 持久性 | 作用 |
|---|---|---|
| 法术熟练度 `masteryLevel` | 永久 | 决定带入局内的初始等级/上限 |
| 当局法术等级 `runLevel` | 单局 | 由顿悟升级，当局结束后清空 |

规则：

```text
如果局外装备五雷正法 masteryLevel 2，则进局 runLevel = 2。
如果空槽进局，局内抽到五雷正法，则 runLevel = 1。
如果局内又选到五雷正法 +1，则 runLevel += 1。
本局结束后，runLevel 不转化为 masteryLevel。
永久升级必须消耗法术残页/玉简/灵石在藏经阁研法。
```

---

### 6.3.3 空槽策略

如果玩家局外不带满 4 个法术：

```text
空槽越多，局内顿悟抽到新法术的权重越高。
```

这形成两种玩法：

| 策略 | 特点 |
|---|---|
| 带满 4 法 | 开局稳定，适合攻坚 |
| 带 1–2 法 | 肉鸽空间更大，适合刷资源/赌连携 |
| 空槽挑战 | 高风险高收益，v0.2 可加成奖励 |

## 7. 聚灵阵：挂机与灵脉

### 7.1 聚灵阵职责

聚灵阵把现实时间转化为局外资源：

```text
修为
下品灵石
少量机缘事件
```

v0.1 不做复杂放置建筑队列，只做单个核心挂机建筑。

---

### 7.2 聚灵阵等级

| 等级 | 修为/min | 灵石/min | 离线容量 | 升级材料 |
|---:|---:|---:|---:|---|
| 1 | 1.2 | 0.8 | 4h | 初始 |
| 2 | 1.8 | 1.1 | 6h | 灵石 + 灵脉之种 x1 |
| 3 | 2.6 | 1.6 | 8h | 灵石 + 灵脉之种 x2 + 灵玉 |
| 4 | 3.8 | 2.4 | 10h | v0.2 |
| 5 | 5.5 | 3.4 | 12h | v0.2 |

### 7.3 离线收益公式

```ts
elapsedMinutes = min(now - lastClaimAt, offlineCapMinutes)
baseCultivation = elapsedMinutes * formation.cultivationPerMinute
baseSpiritStones = elapsedMinutes * formation.spiritStonePerMinute
methodBonus = sum(activeMethodIdleBonuses)
veinBonus = spiritVeinBonus
finalCultivation = baseCultivation * (1 + methodBonus + veinBonus)
```

离线收益不应太强，不能替代局内资源获取。它主要推动“修为”和基础灵石。

### 8.4 丹毒规则

局内消耗丹不产生永久丹毒。永久属性丹产生丹毒。

丹毒分三类：

| 类型 | 来源 | 影响 |
|---|---|---|
| 体丹毒 | 洗髓丹 | 生命恢复效率下降 |
| 气丹毒 | 益气丹 | 真元回复下降 / 法术冷却上升 |
| 神丹毒 | 凝神丹 | 暴击收益递减 / 锁敌抖动 |

v0.1 简化为总丹毒：

```ts
pillToxinTotal = bodyToxin + qiToxin + shenToxin
```

阈值：

| 总丹毒 | 状态 | 局内影响 |
|---:|---|---|
| 0–79 | 清净 | 无 |
| 80–159 | 微毒 | 真元回复 -3% |
| 160–279 | 郁毒 | 真元回复 -6%，法术冷却 +3% |
| 280–399 | 丹壅 | 真元回复 -10%，法术冷却 +6%，丹药消化 +8% |
| 400+ | 药障 | v0.1 不建议允许，需强提示清毒 |

清毒方式：

```text
清心草 x5 + 灵石 x100 → 清丹毒 -50
灵泉碎片 x1 + 灵石 x180 → 清丹毒 -120
```

### 8.5 永久丹效果递减

同类永久丹越吃越弱。

公式：

```ts
actualGain = baseGain * max(0.35, 1 - samePillTakenCount * 0.08)
```

v0.1 为了避免小数属性，可按“进度条”处理：

```text
洗髓丹提供 100 点洗髓进度。
精提升所需进度随次数增加。
```

## 15. 第一版验收标准

### 15.1 回府闭环

一次第一大阶段结算后：

```text
资源正确入库
洞府显示本次收获
聚灵阵可领取收益
玩家可以炼制至少一种丹药
玩家可以进行至少一种功法/法术升级
玩家可以调整下局 Loadout
玩家可以再次进入局内
```

### 15.2 成长反馈

玩家在 2–3 次回府后，应明显感受到：

```text
开局普攻更强
真元/生命更高
法术更好用
丹药准备更充分
局内压力降低或流派更清晰
```

### 15.3 双轨不混淆

UI 和数据中必须保持：

```text
灵气经验 = 局内顿悟
修为 = 境界/突破
```

局外聚灵阵不产出局内灵气经验。

### 15.4 资源不争抢

在线双人结算后：

```text
P1/P2 各自拿到自己的结算 Receipt。
双方资源不会因为队友拾取而减少。
```

## 16. v0.1 结论

局外洞府 v0.1 的核心不是“经营复杂”，而是保证玩家形成这个心流：

```text
我刚刚死了，但我带回了一堆东西。
我现在立刻能变强。
我知道下一局要试什么流派。
我想马上再打一局。
```

最小闭环必须先打通：

```text
结算 → 收菜 → 修功/研法 → 炼丹/炼器 → 配装 → 再进局
```

只要这条链成立，后续弟子、宗门、交易、灵田、魂修转生等系统都可以在不推翻地基的情况下扩展。
