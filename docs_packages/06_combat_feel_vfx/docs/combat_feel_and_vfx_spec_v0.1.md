# 《战斗手感与特效规范 v0.1》

项目：双人雷霆战机修仙版  
版本：v0.1  
目标：为 Codex 实现第一版 Canvas 战斗表现层提供明确、可测试、可数据化的规范。  
适用范围：第一大阶段「青云山·妖潮初临」、第一批本命法宝、灵宝、法术、丹药、Boss、局内雷劫。

---

## 0. 文档定位

本文件不是美术设定集，而是**战斗手感与效果实现契约**。

v0.1 的目标是让第一版可玩 Demo 达到以下效果：

```text
移动干净、判定清楚、敌弹醒目、法术华丽、雷劫压迫、Boss 有仪式感、双人协作一眼可读。
```

所有效果都必须遵守一条最高原则：

> **视觉爽感不能破坏 STG 可读性。**

因此本规范优先解决：

1. 玩家移动与输入反馈。
2. 判定点和敌弹可读性。
3. 法宝/法术/丹药/雷劫的效果层级。
4. 粒子、发光、震屏、闪白的预算。
5. 不依赖外部图片、外部字体、外部音频、外部 CDN。
6. 表现层与确定性模拟的边界。

## 1. 总体体验目标

### 1.1 手感关键词

第一版局内手感应接近：

```text
清脆、精准、高压、华丽、可控、修仙感强。
```

不追求：

```text
物理惯性重、相机漂移强、屏幕特效糊成一团、击中反馈靠大面积遮挡。
```

### 1.2 STG 优先级

在任何高压战斗中，玩家看到信息的优先级必须是：

| 优先级 | 信息 | 说明 |
|---:|---|---|
| 1 | 自机判定点 | 玩家必须始终知道自己会不会死 |
| 2 | 雷劫/秒杀级预警 | 动态环境高压，优先级高于普通弹幕 |
| 3 | 敌弹 | 普通 STG 生死信息 |
| 4 | Boss 大招预警 | 环形弹、横扫、锁定线 |
| 5 | 玩家位置/队友状态 | 双人协作、救援、神魂 |
| 6 | 敌人轮廓 | 清怪目标与威胁来源 |
| 7 | 玩家法术效果 | 爽感，但不可遮挡敌弹 |
| 8 | 掉落物与吸附轨迹 | 奖励反馈，低于战斗危险信息 |
| 9 | 背景粒子、星屑、浮字 | 氛围，最低优先级 |

## 2. 移动与输入手感

### 2.1 移动基准

v0.1 使用纯键盘/手柄 STG 操作，不使用鼠标瞄准。

| 项 | 推荐值 | 说明 |
|---|---:|---|
| 普通移速 | 400 px/s | 适配 1080p 中央战斗区 |
| 专注模式移速 | 235 px/s | 普通速度约 58.75% |
| 斜向移动 | 归一化 | 不能斜向更快 |
| 输入响应 | 1 frame 内 | 按下方向后下一逻辑帧响应 |
| 边界夹紧 | 立即夹紧 | 不允许越界后被弹回 |
| 移动惯性 | 0 或极低 | STG 精密走位优先 |

正式实现建议：

```ts
const normalSpeed = 400;
const focusSpeed = 235;
const diagonalNormalize = true;
const movementInertia = 0;
```

### 2.2 专注模式

专注模式的作用：

1. 降低移速，便于精细擦弹。
2. 显示判定点。
3. 可让部分本命法宝弹道略微聚拢。
4. 不改变真实判定半径。

视觉规则：

```text
按住专注键时：
- 玩家外轮廓透明度降低 15%
- 核心判定点显示为 7px 白金/青白小圆
- 判定点外出现 14px 极淡环
- 玩家弹幕聚拢时使用细线提示，不使用大面积光幕
```

### 2.3 受击反馈

玩家受击时：

```text
0–3 frame：判定点闪白
0–8 frame：玩家外轮廓红/白交替
0–12 frame：小震屏，强度不超过 4px
0–30 frame：无敌闪烁，如果是普通受击
```

死亡时：

```text
肉身破碎 → 短促白闪 → 神魂轮廓出现 → 聚灵阵提示
```

禁止：

```text
受击时全屏大红遮罩超过 8 frame
受击时隐藏判定点
受击时强震屏导致玩家无法判断下一颗弹
```

## 3. 判定点规范

### 3.1 判定点尺寸

| 项 | 值 |
|---|---:|
| 基础判定半径 | 7 px |
| 专注模式显示半径 | 7 px |
| 专注模式外环半径 | 14 px |
| 神识是否影响判定点 | 否 |

神识影响拾取、索敌、暴击、穿透，不影响判定点大小。

### 3.2 判定点显示

普通模式：

```text
判定点默认不强制常亮，但玩家核心必须有明确亮点。
```

专注模式：

```text
判定点必须常亮，绘制在最高 gameplay 层，仅低于 UI overlay。
```

死亡/复活无敌：

```text
判定点变为虚线环或半透明脉冲，明确告诉玩家暂时不可被击中。
```

### 3.3 不能遮挡判定点的效果

以下效果不得覆盖判定点 24px 内圈：

```text
法术爆炸实心填充
掉落吸附流光
浮字
大面积屏幕闪白
玩家弹幕拖尾
Boss 死亡爆炸
雷劫造化清气
```

可以覆盖但必须半透明且在判定点下层：

```text
八卦剑阵环
护盾外圈
聚灵阵救援圈
丹药 Buff 光环
```

## 4. 渲染层级

### 4.1 层级总表

| 层级 | 名称 | 内容 | 规则 |
|---:|---|---|---|
| 0 | `background_far` | 星空、云雾、远山 | 不得闪烁干扰读弹 |
| 10 | `background_near` | 地面卷轴、灵脉纹理 | 低饱和、低对比 |
| 20 | `pickup_trails` | 掉落吸附轨迹 | 低于敌弹 |
| 30 | `player_projectiles_low` | 普通飞剑、葫芦火苗 | 可被敌弹压住 |
| 40 | `enemies` | 小怪、精英 | 轮廓清楚 |
| 50 | `boss` | Boss 本体 | 入场时可高亮 |
| 60 | `player_projectiles_high` | 高价值法术核心线条 | 不得盖住敌弹核心 |
| 70 | `enemy_bullets` | 普通敌弹 | 高对比，最高危险层之一 |
| 80 | `tribulation_warnings` | 雷劫红圈/锁定柱 | 高于普通敌弹 |
| 90 | `tribulation_strikes` | 天雷实体 | 不可被普通特效盖住 |
| 100 | `players` | 玩家与队友 | 判定核心要清楚 |
| 110 | `player_hitbox` | 专注模式判定点 | 几乎最高 gameplay 层 |
| 120 | `rescue_and_soul` | 聚灵阵、神魂 | 不盖住敌弹核心 |
| 130 | `foreground_effects` | 浮字、碎片、清场余波 | 必须低 alpha |
| 200 | `hud` | 左右 UI、Boss 血条 | 不覆盖核心战斗区 |
| 300 | `major_overlay` | 顿悟、暂停、结算 | 战斗暂停时使用 |

### 4.2 敌弹层规则

敌弹必须满足：

```text
白色或高亮核心 + 有色外圈
最小核心半径 >= 3px
普通敌弹 alpha >= 0.92
外圈 glow 不超过弹体半径 2.5 倍
在玩家法术爆炸上方绘制，或爆炸在敌弹处自动挖空
```

### 4.3 玩家法术层规则

玩家法术可以华丽，但要遵守：

```text
大范围填充 alpha <= 0.45
大范围光幕 duration <= 24 frame
敌弹附近自动降低 alpha 或开洞
爆炸核心不覆盖敌弹白芯
法术特效默认低于 enemy_bullets 层
```

## 6. 粒子系统规范

### 6.1 粒子分类

| 类别 | 用途 | 是否进入 gameplay hash |
|---|---|---|
| `visual_particle` | 爆炸、星屑、火花 | 否 |
| `gameplay_marker` | 雷劫预警、Boss 攻击范围 | 是，或由 gameplay event 驱动 |
| `pickup_trail` | 掉落吸附轨迹 | 否，但掉落物位置是 gameplay |
| `hit_spark` | 命中反馈 | 否 |
| `spell_core` | 法术主要判定表现 | 由 gameplay event 驱动 |

### 6.2 粒子预算

| 质量档 | 普通粒子上限 | 法术粒子上限 | 背景粒子上限 | 掉落轨迹上限 | 目标 FPS |
|---|---:|---:|---:|---:|---:|
| Low | 350 | 120 | 80 | 60 | 60 |
| Medium | 800 | 280 | 160 | 120 | 60 |
| High | 1400 | 480 | 260 | 180 | 60 |

v0.1 默认使用 Medium。

### 6.3 粒子生命周期

推荐生命周期：

| 粒子 | 生命周期 |
|---|---:|
| 普通命中火花 | 8–16 frame |
| 小怪死亡碎片 | 18–28 frame |
| 精英死亡爆炸 | 35–55 frame |
| Boss 阶段转化 | 60–90 frame |
| 雷劫电弧 | 8–18 frame |
| 掉落吸附流光 | 根据吸附时间，最多 60 frame |

### 6.4 粒子形状

不依赖外部贴图，全部使用 Canvas 程序绘制：

```text
圆点
短线
菱形
飞剑三角形
环形波纹
分叉雷线
火焰二次贝塞尔曲线
八卦/符文简化线框
```

禁止：

```text
外部 PNG 爆炸贴图
外部字体图标
外部粒子库 CDN
复杂 SVG 图片资源
```

## 7. 屏幕震动规范

### 7.1 震屏分级

| 等级 | 强度 px | 时长 frame | 用途 |
|---|---:|---:|---|
| `micro` | 1–2 | 4–6 | 普通命中、拾取小奖励 |
| `small` | 3–4 | 6–10 | 玩家受击、小怪连爆 |
| `medium` | 5–8 | 10–18 | 精英死亡、重印砸落 |
| `large` | 9–14 | 18–30 | Boss 阶段转换、大法术 |
| `ultimate` | 15–20 | 30–45 | Boss 死亡、最终雷罚 |

### 7.2 震屏规则

```text
震屏只作用于 render camera，不改变 simulation 坐标。
高压躲弹时，普通击杀不触发超过 small 的震屏。
雷劫落雷前 0.5 秒禁止震屏，避免干扰预警读法。
Boss 死亡可强震，但敌弹已清场后再触发。
```

### 7.3 震屏合成

多种震屏同时触发时：

```ts
finalIntensity = max(existingIntensity * decay, newIntensity);
finalDuration = max(existingRemaining, newDuration);
```

不累加，避免连锁爆炸时相机失控。

## 8. 闪白、暗角、色调滤镜

### 8.1 闪白

| 场景 | 白闪 alpha | 时长 |
|---|---:|---:|
| 小怪死亡 | 不使用全屏白闪 | - |
| 精英死亡 | 0.18 | 4 frame |
| 法术释放 | 0.10–0.18 | 3–6 frame |
| 袖里乾坤反击 | 0.22 | 5 frame |
| Boss 死亡 | 0.65 | 8 frame + 余波 |
| 雷劫突破成功 | 0.72 | 10 frame + 金光渐隐 |

全屏闪白必须在敌弹清理后或暂停状态使用。敌弹仍活跃时，全屏白闪 alpha 不得超过 0.22。

### 8.2 暗角

暗角用于：

```text
雷劫降临
Boss 大招蓄力
神魂出窍
顿悟时停
```

| 场景 | 暗角 alpha |
|---|---:|
| Boss 预警 | 0.20–0.30 |
| 雷劫天象异变 | 0.45–0.65 |
| 神魂出窍 | 0.35 |
| 顿悟时停 | 0.65，但战斗已暂停 |

### 8.3 色调滤镜

色调滤镜必须是 render-only：

```text
燃血丹：暖色 saturation +12%，边缘赤光
清心丹：青白扫光，负面图标破碎
时停/顿悟：低饱和 + 轻微蓝灰
雷劫：降低背景亮度，危险预警保持高亮
```

## 9. 音效原则，v0.1 程序音频

v0.1 不引入外部音频资源。推荐使用 WebAudio 合成基础音效。

音效类型：

| 类型 | 声音语言 |
|---|---|
| 飞剑发射 | 短促金属高频 `ping` |
| 葫芦火息 | 低频噪声 + 火焰包络 |
| 重印砸落 | 低频 `thump` + 短噪声 |
| 五雷正法 | 高频电弧 + 连锁短促 tick |
| 八卦剑阵 | 循环轻铃 + 剑气切割 |
| 袖里乾坤 | 低频吸入 + 反击 burst |
| 丹药吞服 | 陶瓷/鼎炉短声 + 气泡 |
| 雷劫预警 | 低频 rumble + 尖锐上升音 |
| 天雷落下 | 白噪 burst + 低频冲击 |
| 顿悟 | 时间冻结 whoosh + 清亮钟声 |

音效同样分优先级，雷劫、玩家受击、Boss 大招、顿悟高于普通击杀。

## 10. 本命法宝效果规范

### 10.1 青霜飞剑

效果关键词：

```text
清脆、细线、穿透、白金剑芒、低遮挡。
```

表现：

```text
飞剑弹体：细长菱形或剑形
拖尾：8–12 frame，青白半透明
命中：小白星 + 2–4 个金色碎屑
穿透：敌人身后出现细线残影
```

不可做成：

```text
大面积激光常亮光幕
覆盖整条路径的高 alpha 白带
```

### 10.2 紫阳葫芦

效果关键词：

```text
扇形、火舌、余烬、清怪爽感。
```

表现：

```text
火息由 5–7 条半透明火舌构成
外缘使用橙红，内缘使用金白
灼烧敌人有小余烬粒子，不增加屏幕常驻火雾
爆燃时先出现 1 frame 小白芯，再扩散橙色环
```

敌弹可读性规则：

```text
火息在 enemy_bullets 层下方。
敌弹穿过火息时，敌弹白芯不被混色。
```

### 10.3 玄岳重印

效果关键词：

```text
重压、预警、砸落、震波。
```

流程：

```text
0–18 frame：目标区域出现金色方印虚影和地面圆环
18 frame：重印砸落
18–32 frame：冲击环扩散，轻微震屏
32–55 frame：尘光淡出
```

可读性规则：

```text
砸落预警低于雷劫预警。
重印冲击波不遮挡敌弹。
```

## 11. 法术效果规范

### 11.1 五雷正法

效果目标：连锁爆发爽感。

流程：

```text
0–4 frame：玩家前方聚雷，细小电弧
4–8 frame：第一道雷击命中目标
8–32 frame：连锁跳跃，每跳 4–6 frame
击杀时：小雷暴环，半径 85px
```

视觉语言：

```text
雷线用折线，不用粗实心光柱。
每次连锁只闪击目标和路径，路径残影 <= 8 frame。
雷暴环使用空心环，不使用实心遮罩。
```

### 11.2 八卦剑阵

效果目标：保命、清弹、救援开路。

流程：

```text
释放瞬间：玩家脚下八卦线框展开
持续 3s：剑阵旋转，外圈 8 把细剑
清弹时：敌弹破碎为 2–3 个小碎光
结束：外圈收束为一点消失
```

可读性规则：

```text
剑阵外圈 alpha <= 0.38
敌弹进入剑阵前仍要可见
清弹只在被清除瞬间给短反馈
```

### 11.3 红莲业火

效果目标：预判铺场、怪潮拦截。

流程：

```text
0–12 frame：目标区出现莲纹圆阵
12–240 frame：低矮火焰周期跳动
敌人死亡：触发爆燃小环
结束：火焰向内坍缩，余烬散落
```

可读性规则：

```text
领域边界清楚，内部透明。
不要做满屏火雾。
火焰颜色不得与敌弹红白芯混淆，敌弹始终在其上层。
```

### 11.4 袖里乾坤

效果目标：吸弹反击，弹幕压力转输出。

流程：

```text
0–8 frame：前方展开虚空扇面
8–120 frame：吸收普通敌弹，弹幕轨迹向中心弯曲
120–150 frame：压缩成黑紫核心
150–170 frame：反射剑气爆发
```

可读性规则：

```text
被吸收的敌弹在真正进入吸收范围前必须仍可见。
不可吸收的弹幕必须用红色外框强调。
黑洞区域 alpha <= 0.55。
```

## 12. 丹药效果规范

## 13. 雷劫效果规范

### 13.1 局内雷劫整体流程

```text
0–2s：天象异变，背景变暗，边缘暗角
2–8s：单点锁定雷，1s 预警
8–16s：频率提高，加入横向雷线
16–22s：双雷锁定，压力最高
22–24s：最终雷罚
成功：造化清气，全屏清场，回满状态，突破金光
```

### 13.2 雷劫预警

雷劫预警必须由三层组成：

```text
外圈：半透明红色危险范围
内圈：真实命中半径，细白/血红线
中心柱：向上延伸的锁定光柱，最后 0.25s 高频闪烁
```

| 参数 | 值 |
|---|---:|
| 预警时长 | 60 frame |
| 最后闪烁 | 15 frame |
| 真实命中圈 alpha | 0.95 |
| 外圈 alpha | 0.25–0.35 |
| 是否高于敌弹 | 是 |

### 13.3 天雷落下

落雷由：

```text
1 frame 白芯
3–8 frame 主雷柱
8–18 frame 分叉电弧
18–35 frame 地面焦痕/电弧残留
```

天雷不可被普通清弹法术摧毁，所以视觉上必须和普通敌弹区分。

### 13.4 突破成功

成功表现：

```text
最终雷罚落下
全屏短白闪
敌弹与小怪转化为金白粒子
玩家核心放大一圈
修为条突破动画
屏幕文字：筑基功成 / 境界突破
生命与真元回满
```

注意：突破金光发生在清场之后，允许短暂覆盖战斗区。

## 14. Boss 效果规范

### 14.1 Boss 入场

流程：

```text
屏幕顶部出现阴影
Boss 名称浮现
Boss 血条展开
背景轻微暗角
Boss 从上方进入固定位置
入场结束后恢复战斗亮度
```

禁止：

```text
Boss 名称长时间挡住玩家活动区
Boss 入场时让敌弹不可读
```

### 14.2 Boss 阶段转换

每个阶段转换：

```text
血条短暂锁定
Boss 外轮廓变色
小型环形震波
新的攻击模式预警 1s
```

阶段转换震屏：`medium` 或 `large`，但敌弹仍存在时不能超过 8px。

### 14.3 Boss 死亡

死亡流程：

```text
0–30 frame：Boss 本体裂纹发光
30–60 frame：多点连爆
60–90 frame：核心塌缩
90–120 frame：大爆炸，清场
120–180 frame：材料与灵气喷出
```

Boss 死亡可以华丽，但必须在 gameplay 清场后执行大面积闪白和强震。

## 17. 连锁爆炸节奏

连锁爆炸是核心爽感，但必须控制节奏。

### 17.1 连锁视觉节拍

```text
命中 → 2 frame 命中星芒
击杀 → 6 frame 小爆芯
连锁 → 4–6 frame 跳跃线
次级爆炸 → 12–18 frame 空心环
余波 → 20 frame 粒子散开
```

### 17.2 连锁爆炸限制

| 项 | v0.1 上限 |
|---|---:|
| 同一帧可触发小爆炸 | 24 |
| 同一帧可触发连锁线 | 18 |
| 同屏连锁余波环 | 30 |
| 同屏爆炸粒子 | 280，Medium 档 |

超过上限时，合并为：

```text
区域群爆 ring + 少量代表性粒子
```

不要让每只小怪都独立生成 50 粒子。

## 19. 表现层与确定性模拟边界

### 19.1 EffectEvent

玩法系统只发事件：

```ts
EffectEvent {
  frame,
  id,
  effectId,
  position,
  attachedEntityId?,
  intensity?,
  seed?,
  tags?
}
```

表现层根据事件和 `visualRng` 生成粒子。

### 19.2 不进入 StateHash 的内容

```text
粒子位置
屏幕震动偏移
浮字位置
背景星屑
音效播放状态
CSS 动画
全屏滤镜渐变
```

### 19.3 进入 StateHash 的内容

```text
雷劫预警真实位置
法术真实判定半径
清弹范围
敌弹位置
玩家位置
掉落物位置
救援进度
丹药真实剩余帧
```

### 19.4 视觉随机

视觉随机使用 `visualRng`，但不影响 gameplay。

```text
同一个 gameplay event 可以在不同客户端渲染不同粒子分布，只要 gameplay state 一致。
```

如需录像回放完全一致，可固定 visualRng seed。

## 20. 低配模式

Low 模式自动降级：

```text
关闭背景星屑颜色变化
粒子上限降到 350
爆炸粒子改为环形 sprite-like Canvas path
关闭多数浮字，只保留关键状态
法术大范围填充改为线框
屏幕震动保留，但滤镜禁用
WebAudio 同时声部限制到 12
```

自动触发条件：

```text
连续 180 frame 平均 FPS < 52
或粒子池长期占用 > 90%
```

恢复条件：

```text
连续 600 frame 平均 FPS > 58
```

## 23. 验收标准

### 23.1 手感验收

| 指标 | 标准 |
|---|---|
| 移动响应 | 按键后 1 frame 内响应 |
| 专注模式 | 判定点清楚，移速降低明显 |

| 法术释放 | 按键后 3 frame 内有明确视觉反馈 |
| 丹药吞服 | 吞服、炼化、生效、结束四态清楚 |

### 23.2 可读性验收

| 场景 | 标准 |
|---|---|
| 1-4 怪潮 | 敌弹仍然清楚，玩家判定点不丢失 |
| 五雷连锁 | 爽但不盖住雷劫/敌弹 |
| 红莲业火 | 火场不把敌弹染没 |
| 袖里乾坤 | 可吸/不可吸弹幕区别明确 |
| 局内雷劫 | 预警圈 1 秒内一眼可读 |
| 双人救援 | 聚灵阵和敌弹都能看清 |

### 23.3 性能验收

| 指标 | 标准 |
|---|---:|
| 1080p Medium | 60 FPS |
| 1-4 高压粒子峰值 | 不低于 55 FPS |
| Boss 死亡爆炸 | 短时不低于 50 FPS |
| Low 模式 | 全程 60 FPS 优先 |
| 内存 | 粒子对象池稳定，不持续增长 |

### 23.4 工程验收

```text
src/sim/** 不创建粒子
src/sim/** 不触发音效
src/sim/** 不读取 Canvas/DOM
Gameplay 效果只发 EffectEvent

不依赖外部 CDN、图片、字体、音频
```

## 24. v0.1 结论

这份规范把第一版战斗表现的方向定为：

```text
精密 STG 读弹优先
修仙法术华丽但受控
雷劫和 Boss 有仪式感
粒子与震屏有预算
所有效果数据驱动
表现层不污染确定性模拟
```

第一版可玩性的关键不只是“特效多”，而是：

> **玩家在满屏法术和雷劫里，仍然能准确判断下一帧该往哪里走、该按哪个法术、该不该吃丹药、该不该救队友。**
