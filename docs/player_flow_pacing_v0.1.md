# 玩家流程与节奏规格 v0.1

## 1. 目标流程

```text
CharacterCreationScreen
  → Confirm Life
  → LifeSimulationScreen
  → Auto monthly flow
  → Major choice pause
  → Optional interlude
  → Stage summary
  → Adult node candidate
```

本流程要求“玩家可理解、可中断、可恢复、可加速”。

## 2. 模拟时间与真实时间

| 世界时间 | 首版展示方式 | 玩家操作 |
|---|---|---|
| 每月 | 自动日志 / 可折叠 | 无需操作 |
| 每半年 | 重大选择 | 必须操作 |
| 每阶段结束 | 阶段总结 | 确认继续 |
| 插曲候选 | 玩法/自动推演选择 | 必须操作 |
| 成年节点 | 总结/路径选择 | 必须操作 |

## 3. 首版时长预算

| 模块 | 标准耗时 |
|---|---:|
| 进入模拟 / 初始说明 | 30–45 秒 |
| 婴幼年 0–3 岁 | 60–90 秒 |
| 童年 4–8 岁 | 2–3 分钟 |
| 少年 9–13 岁 | 2–3 分钟 |
| 青春 14–17 岁 | 3–4 分钟 |
| 成年节点总结 | 45–90 秒 |
| 合计 | 8–12 分钟 |

## 4. 互动密度

建议首版完整流程：

```text
重大选择：18–24 次显式停顿
阶段总结：4 次
玩法插曲候选：4–8 次
实际手动插曲：1–3 次
自动推演插曲：0–5 次
```

不推荐 36 次半年选择全部完整展示。可以将低权重半年节点合并为“平稳半年”或“轻选择”。

## 5. 停顿规则

必须停顿：

```text
重大选择
玩法插曲候选
阶段总结
成年节点
隐藏半揭示
重要伤病/心结产生
```

不必停顿：

```text
普通成长事件
平稳日常
轻微属性变化
无后续 hook 的 breath 事件
```

## 6. 加速与跳过

按钮：

```text
暂停 / 继续
播放到下一选择
播放到阶段总结
快速推演
展开完整日志
```

调试模式：

```text
runToNextMajorChoice()
runToNextStageSummary()
runToAge18()
```

## 7. 插曲节奏

手动玩法插曲应满足：

```text
明确原因
明确风险
明确奖励方向
短小
可自动推演
失败不终止人生
结果回写
```

首版建议只有一个强插曲必须做得可玩：

```text
雨夜后山 STG 或 天外战鼓 STG
```

其余可以先自动推演。
