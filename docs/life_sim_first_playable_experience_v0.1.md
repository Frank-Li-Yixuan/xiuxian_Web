# 18 年人生模拟首版可玩体验规格 v0.1

## 0. 定位

本规格把 0–18 岁人生模拟定义为玩家从“创建角色”进入“正式修行人生”的第一段可玩体验。它不是纯文本剧情，也不是单独小游戏合集，而是一个由命盘驱动的修真人生推演器。

玩家在创建角色页确认“此生”后，系统应进入人生模拟：

```text
创建角色 / 推演天命
  → 确认此生
  → 0–18 岁人生模拟
  → 月度事件自动流动
  → 每半年重大选择
  → 关键选择可能触发玩法插曲
  → 阶段转化 / 入道节点
  → 成年节点候选
  → 后续试炼 / 洞府 / 系统家园
```

首版可玩体验要让玩家明确感受到：

```text
我的命盘决定了会遇到什么。
我的天命不是卡牌，而是在人生里发生。
我的半年选择会改变成长方向。
我的随身物和隐藏预兆在逐渐显化。
我触发的玩法插曲不是小游戏乱入，而是命运关键节点。
这段人生会影响后续战斗和局外修行。
```

## 1. 首版目标

首版不追求完整 216 个月“每月点一下”的厚度，而追求一段能完整跑通、能截图、能调参、能让玩家愿意重开的体验。

### 1.1 首次游玩时长

建议首版完整 0–18 岁模拟时长：

```text
首次正常体验：8–12 分钟
熟悉后快速体验：4–6 分钟
Debug/开发快速推演：20–60 秒
```

如果超过 15 分钟，玩家会觉得“进正式游戏太慢”。如果低于 3 分钟，人生缺少沉淀。

### 1.2 玩家输入频率

虽然世界内有 216 个月，但玩家不应操作 216 次。

首版推荐：

```text
月度事件：自动播放，按阶段聚合展示
半年选择：36 次理论节点，但首版可按“半年选择窗口”表现为 18–24 个主要选择
玩法插曲：完整人生 4–8 次，其中强制/主线插曲 1–3 次
阶段总结：每个年龄阶段至少 1 次
```

首版可以把轻事件压缩成“月度流”，只在重要节点停顿。

## 2. 首版可玩流程

### 2.1 创建角色后进入人生模拟

输入：

```ts
CharacterOriginState
OpeningInnateDraft
DestinySelectionState
OriginFateDraft
```

进入人生模拟时生成：

```ts
LifeSimulationPlayableState
```

其中包含：

```text
当前年龄月
当前年龄阶段
当前修行身份阶段
九宫命盘状态
人生主线状态
事件线状态
隐藏命叙事状态
随身物生命周期
月度日志
半年选择状态
插曲结果
节奏预算
```

### 2.2 自动月度流

月度事件不弹窗打断。默认播放为一个可滚动/自动流动的日志流：

```text
一岁三月：窗外雷雨大作，你整夜不哭，只盯着屋梁缝隙。
一岁四月：母亲说你比旁的孩子安静许多。
一岁五月：祖传玉佩在襁褓边微温。
```

系统每月推进时：

```text
1. 读取当前 LifeSimulationState
2. 读取 NarrativeDensityWindow
3. 选择月度事件
4. 应用可见/隐藏效果
5. 写日志
6. 累积 hooks
7. 检查是否到半年选择节点
```

### 2.3 半年重大选择

每 6 个月或关键窗口暂停一次，展示：

```text
标题
事件描述
过去几个月相关日志摘要
3–4 个选择
每个选择的风险标签
可见收益倾向
隐藏/命格/随身物提示
是否会进入玩法插曲
```

示例：

```text
【七岁·雨夜后山】
暴雨夜，后山传来低沉雷鸣，像有人在天边敲鼓。父母劝你不要出门。

[稳] 留在家中，闭门读书。
[险] 偷偷前往后山查看。  可能触发：灵识试炼
[命] 迎着雷声打坐，任那声音靠近。  雷声似乎离你太近了
```

### 2.4 玩法插曲

半年选择可以产生 `interludeCandidate`。首版建议只做插曲入口协议和 1–2 个可运行插曲，其他可走自动推演。

首版推荐必须支持：

```text
STG 短插曲：雨夜后山 / 天外战鼓
文本判定插曲：闭门百日 / 无师吐纳
DBG/割草/阵法：允许作为 planned / auto-resolve，不强制完整做完
```

插曲结果必须回写：

```text
主线分数
事件线进度
伤病 / 心结
功德 / 业力
隐藏命进度
随身物亲和
阶段转化 token
age18 hook
```

### 2.5 阶段总结

每个年龄阶段结束时，展示阶段总结。

阶段总结应包括：

```text
本阶段主要经历
九宫变化摘要
显著天命显化
身世/随身物/隐藏预兆进展
主要选择后果
当前主线倾向
```

示例：

```text
【童年小结 · 八岁】
这些年你多在药铺与后山之间来回。你逐渐能分辨几味草药的寒热，也时常梦见一尊残破丹炉。雨夜后山那次之后，你听见雷声时会短暂停住脚步。

变化：药理 +7，灵感 +4，心结：惧雷初显，随身物：药铺铜炉亲和加深。
```

## 3. 首版必须做什么

### 3.1 必须实现

```text
1. LifeSimulationPlayableState
2. LifeSimulationScreen DOM 页面
3. 自动月度播放
4. 每 6 个月暂停 major choice
5. major choice 选择结算
6. 至少 4 个年龄阶段的阶段总结
7. 主线分数显示
8. 隐藏预兆不泄露真名
9. 随身物亲和显示
10. 玩法插曲入口协议
11. 至少一个 STG 短插曲可手动进入或用现有战斗壳运行
12. 插曲结果回写
13. 加速 / 暂停 / 自动推进
14. 存档恢复
15. 216 月结束后的成年节点占位或跳转 Age18 pending
```

### 3.2 首版可暂缓

```text
完整 DBG 玩法插曲
完整割草玩法插曲
完整天地棋局插曲
真实 DeepSeek API 接入
全部 216 个月逐月高质量文本
所有隐藏命完整揭示
复杂 NPC 关系网
宗门拜师完整剧情
18 岁之后所有正式模式
```

## 4. 首版页面结构

### 4.1 LifeSimulationScreen 布局

建议 1920×1080：

```text
┌──────────────────────────────────────────────────────────────┐
│ Header：人生模拟 / 年龄 / 阶段 / 当前身份 / 速度 / 暂停        │
├───────────────┬──────────────────────────────┬───────────────┤
│ 左：命盘状态   │ 中：事件流与当前重大选择       │ 右：主线/随身物 │
│ 九宫变化       │ 月度日志                     │ 主线进度        │
│ 天命显化       │ 半年选择卡                   │ 事件线状态      │
│ 隐藏预兆       │ 插曲入口                     │ 随身物亲和      │
├───────────────┴──────────────────────────────┴───────────────┤
│ Bottom：时间轴 0–18 岁 / 阶段节点 / 已发生玩法插曲 / 操作按钮  │
└──────────────────────────────────────────────────────────────┘
```

1366×768：

```text
Header 固定
中间事件流 + 当前选择为主
左/右信息面板折叠为 Tabs
底部时间轴缩小
```

### 4.2 信息优先级

玩家最需要看到：

```text
现在几岁
为什么停下来让我选
这些选择大概有什么风险
过去几个月发生了什么
这个选择是否会触发玩法
我的命盘/天命/身世有何影响
选择后发生了什么
```

不需要每秒盯着全部数值。

## 5. 首版节奏模型

### 5.1 播放速度

提供三档：

```text
慢速：每月日志 1.2s
标准：每月日志 0.45s
快速：每月日志 0.12s
```

半年选择暂停，等待玩家确认。

### 5.2 自动推进模式

```text
自动播放到下一次重大选择
自动播放到阶段结束
自动播放到 18 岁
```

但若触发玩法插曲候选，必须停下来让玩家选择：

```text
手动挑战
自动推演
暂不处理/返回选择
```

### 5.3 快速体验压缩

为了避免 216 条日志太长，系统可以按显示策略压缩：

```text
普通 breath 事件可以折叠为“平稳数月”
重要 omen / thread / choice_seed / transition_seed 必须单独显示
玩家可展开完整日志
```

## 6. 叙事与规则边界

### 6.1 LLM 使用

首版建议先用 fallback 模板，保留 LLM hook。

如果接 DeepSeek：

```text
只能润色日志、选择文案、阶段总结
不能决定事件、数值、成功失败
不能泄露隐藏真名
必须缓存
失败必须 fallback
```

### 6.2 隐藏信息

页面禁止显示：

```text
古雷真血
丹圣遗骨
系统共鸣体
前世剑魄
魔印微痕
```

除非 revealState 已经达到 `revealed`。

早期只显示：

```text
雷声似乎离你更近
陌生火候
木剑低鸣
耳边杂音
荒祠黑影
```

## 7. 首版成败标准

玩家在 10 分钟体验后，应能说出：

```text
我这个角色大概是什么命。
我为什么会遇到这些事。
我的几次选择改变了人生方向。
我触发的小游戏/试炼不是随机插入，而是事件结果。
我的随身物和隐藏预兆变得更有存在感。
我愿意再 Roll 一个不同命盘试一次。
```

如果玩家只感觉：

```text
我看了一堆随机文本。
我点了很多无意义选择。
我的命盘没有体现。
玩法插曲像强插小游戏。
```

则首版失败。
