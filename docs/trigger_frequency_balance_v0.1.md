# 插曲触发频率与节奏控制 v0.1

## 1. 总体预算

0–18 岁共有 36 次半年选择。建议全程：

```text
玩法插曲 8–12 次
核心主线插曲 2–4 次
其余为文本/判定选择
```

## 2. 首次游玩推荐预算

首次游玩应减少打断：

| 年龄段 | 推荐插曲 |
|---|---:|
| 0–3 | 0 |
| 4–8 | 1–2 |
| 9–13 | 2–3 |
| 14–17 | 3–5 |
| 合计 | 6–10 |

## 3. 冷却规则

同一模式冷却：

```text
同模式玩法插曲至少间隔 12 个月。
核心主线插曲至少间隔 18 个月。
```

同一事件线冷却：

```text
同一 thread 的插曲至少间隔 12 个月。
```

## 4. 插曲疲劳值

```ts
interludeFatigue = recentInterludesLast24Months * 10
```

超过阈值会降低触发权重：

| fatigue | 权重惩罚 |
|---:|---:|
| 0–10 | 无 |
| 11–20 | -20% |
| 21–30 | -40% |
| 31+ | -65% |

## 5. 强制体验点

v0.1 可保留 2 个强制或半强制体验点：

```text
第一次玩法插曲：4–8 岁阶段的一次轻梦境/试炼
青春阶段主线玩法插曲：14–17 岁阶段一次核心试炼
```

但用户可以选择自动结算，不能强迫玩家长期反复战斗。

## 6. 手动 vs 自动结算

| 方式 | 成功上限 | 隐藏触发 | 推荐用途 |
|---|---|---|---|
| 手动玩法 | hiddenSuccess | 可触发 | 高收益 |
| 自动结算 | success | 不触发 hiddenSuccess | 快速推进 |
| 跳过放弃 | partial/failure | 不触发 | 避险 |

## 7. 调参指标

实现后应统计：

```text
平均每生涯插曲次数
各模式出现占比
4–8 / 9–13 / 14–17 插曲分布
玩家手动参与率
自动结算率
插曲失败率
插曲导致伤病率
插曲产生 age18Hook 次数
```

验收目标：

```text
普通角色平均 7–10 次插曲
极端命盘平均 9–12 次插曲
系统前兆线 dominant 时至少 1 次 system_preview 插曲
无角色 0–3 岁出现真实玩法
```
