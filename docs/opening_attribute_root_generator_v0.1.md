# 《开局属性与灵根随机生成器 v0.1》

## 0. 文档定位

本文件定义创建角色阶段的第一层随机系统：

```text
创建存档
  → 创建角色页面
  → 生成开局属性与灵根
  → 天命词条系统读取这些结果并生成主天命/副天命/劫命
  → 隐藏血脉、身世与随身物系统继续补全
  → 进入 0–18 岁人生模拟
```

本系统只负责“先天底子”，不负责完整人生事件，也不负责最终域外战场数值。它输出一个稳定、可复现、可锁定重 Roll、能被后续系统消费的 `OpeningInnateDraft`。

核心目标：

1. **每次重 Roll 都像抽命，不像掷普通属性点。**
2. **属性分布要有戏剧性，有偏科、有废柴、有天才、有福缘、有短命。**
3. **灵根要影响人生模拟、天命权重、局外法脉、局内战斗模式。**
4. **同 seed 可复现，Codex 实现时不能使用 `Math.random()`。**
5. **支持锁定：玩家可锁定灵根/属性命盘后继续重 Roll 其他部分。**

---

## 1. 开局随机的总流程

推荐生成顺序：

```text
GenerateOpeningInnateDraft(seed, rerollState, locks)
  ↓
1. 选择命盘类型 attributeArchetype
  ↓
2. 生成六维先天资质 aptitudeStats
  ↓
3. 生成精气神种子 coreSeedStats
  ↓
4. 生成灵根类型 spiritualRootCategory
  ↓
5. 生成灵根五行向量 rootElementVector
  ↓
6. 计算灵根纯度、稳定性、冲突度
  ↓
7. 生成成长偏置 growthBias
  ↓
8. 生成标签 destinyBiasTags / lifeEventBiasTags / modeBiasTags
  ↓
9. 计算命盘戏剧性 distinctivenessScore
  ↓
10. 如过于平庸，触发一次轻量补偿性“命盘钩子”
  ↓
输出 OpeningInnateDraft
```

注意：
这里的“补偿”不是送强度，而是保证每个角色有记忆点。例如一个普通命盘可能获得“灵感偏高”“寿元偏低”“金木相克灵根”这样的构筑钩子，而不是单纯提升数值。

## 2. 属性分层

### 2.1 先天资质：出生时决定长期潜力

| 属性 | 范围 | 含义 | 主要影响 |
|---|---:|---|---|
| 根骨 `rootBone` | 1–120 | 身体承载、炼体资质 | 人生体魄成长、域外生命、局外修炼效率 |
| 悟性 `comprehension` | 1–120 | 学法悟道能力 | 天命权重、读书/研法事件、顿悟质量 |
| 灵感 `inspiration` | 1–120 | 感应异象、梦境、隐秘能力 | 隐藏血脉、奇遇、异常事件 |
| 气运 `fortune` | 1–120 | 福缘、命数、偶然性 | 掉落、保命、稀有事件、重 Roll 体验 |
| 心性 `heart` | 1–120 | 道心稳定程度 | 抗心魔、突破稳定、成功判定 |
| 寿元 `lifespan` | 1–120 | 命火长度 | 长线修行、借寿、短命天命风险 |

默认 UI 显示为 1–100，但内部允许超过 100，表示“超凡资质”。
低于 20 是明显缺陷，高于 90 是强辨识度。

### 2.2 精气神种子：人生模拟中的可成长底子

| 属性 | 出生范围 | 18 岁转化 |
|---|---:|---|
| 精 `jingSeed` | 6–24 | 生命、承载、抗压 |
| 气 `qiSeed` | 5–24 | 真元、吐纳、法术频率 |
| 神 `shenSeed` | 5–24 | 神识、索敌、拾取、顿悟 |

精气神在出生阶段不应过高。
人生模拟通过每月事件和半年选择把它们逐渐养起来。

### 2.3 成长偏置

`growthBias` 不直接显示给玩家，但会影响 0–18 岁事件收益：

```ts
interface OpeningGrowthBias {
  jingGrowth: number;       // 肉身成长倾向
  qiGrowth: number;         // 真元成长倾向
  shenGrowth: number;       // 神识成长倾向
  studyBias: number;        // 读书/研法事件收益
  martialBias: number;      // 练武/实战事件收益
  alchemyBias: number;      // 药理/炼丹事件收益
  artifactBias: number;     // 器物/法宝事件收益
  seclusionBias: number;    // 闭关/苟道事件收益
  adventureBias: number;    // 探索/冒险事件收益
}
```

## 4. 随机算法规范

### 4.1 禁止使用

```text
Math.random()
Date.now() 参与玩法随机
浏览器真实时间影响生成
UI 层临时随机影响结果
```

### 4.2 必须使用 Seeded RNG

推荐流分离：

```ts
interface OpeningRngStreams {
  archetype: SeededRng;
  aptitude: SeededRng;
  coreSeed: SeededRng;
  spiritualRoot: SeededRng;
  elementVector: SeededRng;
  drama: SeededRng;
}
```

同一个 seed、同样锁定状态、同样重 Roll 次数，结果必须一致。

### 4.3 推荐分布：三骰钟形分布

不要所有值均匀分布。推荐：

```ts
function bell01(rng: SeededRng): number {
  return (rng.nextFloat() + rng.nextFloat() + rng.nextFloat()) / 3;
}
```

然后映射到 `[min, max]`：

```ts
value = round(min + bell01(rng) * (max - min));
```

这样大部分数值靠近中间，极端值更少，更像“资质”。

### 4.4 戏剧性补偿

生成后计算 `distinctivenessScore`：

```text
+2：任一资质 ≥ 90
+1：任一资质 ≤ 20
+2：异灵根 / 隐灵根 / 天灵根
+1：双灵根相生
+1：双灵根相克
+1：寿元 ≤ 25
+1：灵感 ≥ 85
+1：气运 ≥ 85
```

若分数 < 2，则执行一次“轻量命盘钩子”：

```text
随机提高一个非核心特征，例如灵感 +8、气运 +8、心性 +8；
或给灵根增加“轻微异象”标签；
或给寿元轻微偏低，但补偿悟性/灵感。
```

这不是送强度，而是让角色有记忆点。

## 5. 灵根系统

### 5.1 灵根分类

| 类型 | 基础概率 | 说明 |
|---|---:|---|
| 单灵根 | 28% | 方向明确，修炼稳定 |
| 双灵根 | 27% | 可做相生/相克连携 |
| 三灵根 | 15% | 灵活，成长略慢 |
| 杂灵根 | 13% | 初期慢，多模式适配好 |
| 天灵根 | 3% | 单属性极强，路线窄 |
| 异灵根 | 5% | 雷、阴等特殊路线 |
| 隐灵根 | 5% | 表面普通，后续觉醒 |
| 闭塞灵根 | 2% | 前期很差，逆命/隐藏路线 |
| 混沌灵根 | 2% | 极不稳定，高风险稀有路线 |

实际概率会被命盘类型修正。
例如“灵根异象”会提高异灵根、隐灵根、天灵根权重；“废柴逆骨”会提高闭塞灵根和隐灵根权重。

### 5.2 元素

v0.1 使用：

```text
metal 金
wood 木
water 水
fire 火
earth 土
thunder 雷
yin 阴
mixed 杂

```

```text

```

五行基础相克：

```text
木克土

```

```text

杂：宽适配、低纯度、多模式发展
```

### 5.3 灵根纯度、稳定性、冲突度

```ts
interface SpiritualRootMetrics {
  purity: number;       // 0–120，越高越适合专精

  conflict: number;     // 0–120，越高越易出问题，但也有特殊机缘
```

## 9. 与锁定重 Roll 的关系

本系统至少支持锁定：

```text
attributeArchetype
aptitudeStats
coreSeed
spiritualRootCategory
spiritualRootElements
spiritualRootFull
```

推荐 UI 第一版只提供：

```text
锁定属性命盘
锁定灵根
```

如果拆得太细，玩家会被复杂度压垮。

锁定规则：

```text
锁定属性命盘：保留 archetype + aptitude + coreSeed，重 Roll 灵根和后续天命。
锁定灵根：保留 spiritualRoot 完整对象，重 Roll 属性和后续天命。
锁定全部先天：保留本系统输出，只重 Roll 天命。
```
