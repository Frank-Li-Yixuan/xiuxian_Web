# SIM-REDESIGN 总集成 v0.1

## 1. 总目标

当前项目已经从单一“修仙版雷霆战机”升级为：

```text
局外修真模拟器为主体
  + 多模式试炼为验证场
  + 域外战场 STG 为其中一个重要试炼模式
```

因此人生模拟阶段不能再只是“角色创建后的过场”。它必须成为局外主体的第一段核心体验：

```text
新建存档
  → 推演天命 / 创建角色
  → 0–18 岁人生模拟
  → 月度事件与半年选择
  → 关键节点玩法插曲
  → 入道 / 系统候选 / 成年节点
  → 第一场试炼或洞府开启
```

## 2. 总集成原则

### 2.1 命盘先于事件

事件不应该凭空随机，而应从以下结构推导：

```text
九宫命盘
  → 三才 / 阴阳 / 五行 / 派生评分
  → 灵根与天命成立条件
  → 身世、隐藏命、随身物
  → 人生主线与事件线
  → 月度事件与半年选择
```

### 2.2 天命不是 Buff，是命理结果

天命必须有：

```text
成立条件
支撑条件
反相条件
变异规则
人生显化 hooks
玩法投射 hooks
```

不合理组合要变异，不要生硬删除。

### 2.3 人生模拟不是流水账，是主线显化

216 个月不应是 216 条随机日志，而应由：

```text
人生主线
事件线
叙事密度控制
半年关键选择
玩法插曲
阶段转化
```

共同控制节奏。

### 2.4 玩法插曲服务人生主线

STG、割草、DBG、自走棋不应作为随机小游戏乱入，而应作为事件线 crisis/trial 节点的可操作验证。

### 2.5 18 岁不是唯一硬切

18 岁是默认成年节点，但阶段转化应分成：

```text
年龄阶段
修行身份阶段
入道节点
系统候选节点
成年试炼节点
```

### 2.6 LLM 只写文案，不改规则

DeepSeek/LLM 可以生成：

```text
月度日志
半年选择描述
阶段总结
人生小传
```

但绝不能决定：

```text
数值
奖励
成功失败
隐藏血脉进度
天命生成
玩法插曲结果
```
