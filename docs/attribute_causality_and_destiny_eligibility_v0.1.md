# 《属性因果与天命成立条件 v0.1》

## 1. 为什么需要成立条件

强天命必须解释得通。

不合理例子：

```text
悟性 22、灵感 18、神 30、寿元 90
却拥有【天妒英才】
```

这会破坏玩家信任。

合理处理：

```text
如果高天赋不成立，则不应出现天妒英才。
如果确实因天机异常被抽中，应变异为“妄承天机”。
```

---

## 2. 天命三层规则

每个天命必须有：

```text
成立条件
支撑条件
反相条件
```

### 2.1 成立条件

说明这个天命为什么成立。

例如【天妒英才】：

```text
talentScore >= 85
或 悟性 >= 90
或 灵感 >= 90
或 悟性 + 灵感 >= 160
```

### 2.2 支撑条件

让天命更合理、更强、更容易显化。

例如：

```text
寿元低
心性不稳
天道压力高
雷雨/天机/系统前兆标签
```

### 2.3 反相条件

出现则不适合该天命。

例如：

```text
talentScore <= 55
悟性与灵感都低
寿元高、心性高、气运顺
```

---

## 3. 核心天命示例

## 3.1 天妒英才

本质：

```text
天赋太高，命数承载不住。
```

可以低根骨，但不能低悟性/低灵感。

成立：

```text
talentScore 高
悟性/灵感异常高
```

支撑：

```text
寿元低
命火弱
天道压力高
心性不稳
```

变异：

```text
妄承天机
```

---

## 3.2 废灵逆命

本质：

```text
低起点 + 高韧性 + 失败转化。
```

成立：

```text
根骨低
灵根杂/闭塞
心性高
灵感异常
```

反相：

```text
天灵根、高根骨、高悟性、高气运
```

变异：

```text
天骄遭厄
```

---

## 3.3 苟道至尊

本质：

```text
避战、长寿、心性、隐忍、潜修。
```

成立：

```text
心性高
寿元高
lateBloomScore 高
气运不差
```

反相：

```text
以战养战
强杀伐
心性低
```

变异：

```text
隐杀之命
```

---

## 3.4 魔心暗种

本质：

```text
高感应 + 心性裂缝 + 禁忌诱惑。
```

成立：

```text
神高/灵感高
心性偏低
荒祠/守墓/黑骨短笛/魔印标签
```

反相：

```text
清净琉璃心
功德极高
心性极高
```

变异：

```text
净莲藏影
```

---

## 4. 变异命格设计原则

变异命格不是惩罚，而是把矛盾变成故事。

| 冲突 | 普通做法 | 推荐做法 |
|---|---|---|
| 天才条件不足 | reroll 掉 | 妄承天机 |
| 废柴条件不足 | reroll 掉 | 天骄遭厄 |
| 苟道和杀伐冲突 | 互斥删除 | 隐杀之命 |
| 清净和魔心冲突 | 二选一 | 净莲藏影 |
| 丹道但心性低 | 删除丹道 | 躁火炼药 |

这样能保留 Roll 的惊喜。

## 5. UI 呈现

角色创建页面可以显示：

```text
该天命与当前命盘相合
该天命与当前命盘相冲
该天命似有异变
```

不要显示复杂公式，但可以显示一句解释：

```text
你的悟性极高而寿元薄弱，此命似受天道所忌。
```

或者：

```text
你并非真正天才，却被某种天机误标记，此命透着不祥。
```

## 6. 工程要求

Codex 应实现：

```ts
evaluateDestinyEligibility(trait, ninePalaceEvaluation): DestinyEligibilityResult
applyAntiWeirdnessRules(selection): AntiWeirdnessResult
```

并保证：

```text
1. 强天命有因果解释。
2. 不合理组合会变异或警告。
3. 变异结果可复现。
4. 变异结果写入 debug info，供调试页查看。
```
